# 第二阶段增强说明

## 这一轮做了什么

这次修改没有把项目扩成大而杂的版本，而是围绕 Spacewar! 最有代表性的玩法基因，做了一轮适合当前单人波次原型的小步增强：

- 中央重力恒星
- Hyperspace 紧急跃迁
- 冲锋型敌人
- 混编波次
- 对应 HUD / 结果页信息补充

## 设计依据

这轮实现主要参考了原始 Spacewar! 的几个经典元素：

- 恒星重力井会直接影响机动路线
- 玩家可以把“贴星借力”当成高风险高收益手段
- Hyperspace 是典型的 panic button
- Hyperspace 的重入存在随机性，部分版本里还会随着使用次数增加而越来越危险

这些特征非常适合当前项目，因为它们能在不引入复杂剧情或大体量资源的前提下，显著增加战场决策密度。

## 本次具体落地

### 1. 中央重力恒星

战斗场中央加入了一个重力星体：

- 玩家移动会受到轻度引力影响
- 敌人同样会被拉拽
- 贴得太近会持续受伤
- 敌人被拉进核心也会被摧毁

它的作用不是单纯提高难度，而是让战场从“空平面”变成“有风险地形的平面”。

### 2. Hyperspace

新增 `F` 键紧急跃迁：

- 消耗有限次数
- 跃迁到安全随机位置
- 有失败风险
- 使用次数越多，失败概率越高

为了适配当前单人波次原型，这里没有直接沿用“失败即秒杀”的最严厉版本，而是改成“失败时高额受伤”。这是对原始机制的保守改编，目的是保住当前原型的可玩性。

### 3. 冲锋型敌人

新增 `Diver` 敌人：

- 平时会调整站位
- 周期性向玩家发起高速冲锋
- 在有重力星的场地里更容易制造压力点

这样波次不再只是“同一种敌人越来越多”，而是开始出现局部节奏变化。

### 4. 升级联动

新增两种更贴近本轮主题的升级：

- `Hyperfield Tuning`
- `Solar Shield`

它们分别让跃迁和重力星机制变成可以经营的资源，而不只是纯粹的环境惩罚。

## 当前建议的观察重点

如果继续试玩和迭代，建议重点观察这几个问题：

1. 重力星是否真的制造了有趣决策，还是只是在干扰操作。
2. Hyperspace 是否足够像“救命按钮”，但又不会太强。
3. `Diver` 是否让中期波次更有压迫感，而不只是更烦。
4. 升级是否已经开始让不同局之间出现可感知差异。
